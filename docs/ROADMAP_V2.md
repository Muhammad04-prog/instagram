# ROADMAP v2 — Миграция на новый backend (NestJS)

**Новый API:** `https://backend-instagram-kvv4.onrender.com/api`
**Swagger:** `/api/docs-json` — **170 endpoints** против 57 у softclub.
Конверт ответа тот же: `{ data, errors, statusCode }` → `lib/axios.ts` unwrap **остаётся**.

Этот бэкенд написан как **замена** softclub: в его же описаниях полей стоят ссылки на наши баги
(`gender` — «Симметричный enum: что отправили — то и вернётся (баг softclub #12)`). Все 5 🔴-багов
из `BACKEND_BUGS.md` здесь закрыты, плюс появилось то, чего в API не было и что мы честно
не делали: уведомления, запросы на переписку, архив историй, «Добавить место», дата рождения.

Правило прежнее: **одна фаза = одна сессия**, строго сверху вниз, `[x]` по готовности.

## Что ломается (главное)

| Что             | Было (softclub)                     | Стало (NestJS)                                                                                   |
| --------------- | ----------------------------------- | ------------------------------------------------------------------------------------------------ |
| Базовый URL     | `instagram-api.softclub.tj`         | `backend-instagram-kvv4.onrender.com/api`                                                        |
| Роуты           | `/Post/get-posts` (Pascal, глаголы) | `/posts`, `/posts/feed` (REST, ресурсы)                                                          |
| Auth            | 1 access-токен                      | **access + refresh** (ротация) + `role`, `isVerified`                                            |
| Login           | `userName` + `password`             | `login` = userName **ИЛИ** email **ИЛИ** phone                                                   |
| Reset password  | 2 шага (DELETE!)                    | 4 шага: forgot → **код 6 цифр на email** → verify-code → resetToken                              |
| Register        | 5 полей                             | + `dob` (**обязателен**) + `phone` (опц.) → блок «Дата рождения» из img4 **теперь делаем**       |
| id пользователя | `int`                               | **`string` (uuid)** — ломает все типы и роуты                                                    |
| Пагинация       | `page`/`limit`                      | **`cursor`/`limit`** — все infinite-списки                                                       |
| `gender`        | читался `"Male"`, писался `0│1` 🔴  | enum `MALE│FEMALE│OTHER│HIDDEN` симметрично                                                      |
| Медиа поста     | `images: string[]`                  | `media: [{ url, type: IMAGE│VIDEO, width, height, duration, thumbUrl, filter }]`                 |
| Пост            | без автора                          | `author`, `location`, `music`, `taggedUsers`, `hashtags`, `isLiked`, `isFavorited`, `isArchived` |
| Story           | «видел ли я» не отдавал 🔴          | `isViewed`, `isLiked`, `likesCount` — **localStorage-костыль из Фазы 6 удаляем**                 |
| Чат             | без last message / unread 🔴        | `lastMessage`, `lastMessageAt`, `unreadCount`, `isOnline`, `isMuted`, `theme`                    |
| Realtime        | хаба нет → поллинг 5 сек 🔴         | **сокет** (звонки/эфиры сигналятся через него)                                                   |
| Комментарии     | `userName` всегда `null` 🔴         | автор приходит; + лайки и **ответы (ветки)**                                                     |

## Что чинится само (закрываем `BACKEND_BUGS.md`)

`delete-user` 403 → `DELETE /users/me` (soft-delete) · `update-Location` 400 AutoMapper → `PUT /locations/{id}` ·
`delete-message` без OwnerGuard → **OwnerGuard есть** · `delete-user-image-profile` ломал login → `DELETE /profile/avatar` ·
нет unread/realtime → есть оба · нет истории с `createdAt` → **`createdAt` есть**.

> - ⚠️ **БД лежала всю фазу** (`database/redis/storage: down`), поэтому правило «живой API авлотар»

>   поднимут — прогнать регистрацию → логин → ленту и сверить (особенно `/auth/me`: в Swagger он
>   описан как «текущий пользователь + профиль», и `session/route.ts` принимает **оба** варианта —

>   `<Image>` его не отрисует.
> - **Refresh-токены ротируются**, поэтому обновление идёт **проактивно** (по `exp` с запасом 30 сек),
>   а не по 401: тело запроса стримится (multipart), а поток нельзя переиграть — запрос должен быть
>   верным с первого раза. Параллельные запросы делят один промис обновления, иначе проигравший
>   предъявил бы уже отозванный токен и вылетел бы в логаут.

>   («курсор последнего элемента предыдущей страницы» — Swagger, `GET /users`). Короткая страница = конец.
> - ⚠️ **`/stories/{id}/viewers` не пагинируется**: у `StoryViewerDto` нет своего `id`, а курсор — это
>   именно id последней строки. Берём разом лимит 50; больше 50 зрителей — обрежется.

>   Остальное по Фазе 12 (429-тексты, logout-ревокация в UI) — там же.

>   в API просто не было. HEX по-прежнему только в `globals.css`.

> - Проверено вживую (Playwright, dark + light + /ru): `/register` совпадает с img4 (День/Месяц/Год),
