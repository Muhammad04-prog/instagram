## 🔴 Главное: ни один из 113 endpoint'ов не подтверждён

БД бэкенда лежит всю миграцию:

```
GET /api/health → {"database":"down","redis":"down","storage":"down"}
```

Приложение NestJS живое (валидация отвечает 400, guard'ы — 401), но **всё, что идёт в БД, падает
с 500**. Значит все DTO взяты из Swagger и **не сверены с реальностью**.

**Как только БД поднимут — первым делом, до нового кода:** регистрация → логин → лента → пост →
чат, и сверить каждый DTO с живым ответом. Это не бюрократия: на прошлом бэкенде живая проверка
ловила расхождения со Swagger в **8 фазах из 10** (`gender` читался строкой `"Male"`, а на запись
принимал только `0|1`; `get-following-post` без `UserId` молча отдавал пустую ленту).

Отдельно проверить:

- `/auth/me` — Swagger говорит «пользователь + профиль», но не показывает форму.
  `src/app/api/auth/session/route.ts` принимает **оба** варианта (голый `AuthUserDto` и
  `{ user, profile }`). Как узнаем правду — убрать ветку.
- **Хост медиа.** Ни одного настоящего URL мы не видели. В `next.config.ts` только хост API;
  `<Image>` откажется грузить незнакомый домен. Первый живой `avatarUrl` → добавить в
  `remotePatterns`.

## Что заблокировано бэкендом (не делать, пока не ответят)

- **Сокет** (чат real-time). Сам сокет живой — `GET /socket.io/?EIO=4&transport=polling` отдаёт
  handshake, он в **корне домена**, не под `/api`. Но: (1) авторизоваться нечем — токен только в
  httpOnly-cookie, сокет на другом origin, нужен `POST /auth/socket-ticket`; (2) имена событий
  нигде не описаны. Подробности — `BACKEND_REQUEST.md` §4.
- **Звонки** (`POST /chats/{id}/call`) — сигналинг через тот же сокет. Кнопка без него создаст
  звонок в никуда.
- **Live** (Фаза 21, 0/18) — нужен LiveKit SDK + ключи; в Swagger торчит `ws://localhost:7880`.

## Правила, которые здесь дорого стоили

Не из вежливости — каждое куплено ошибкой:

1. **Проверять в браузере, а не читать код.** Так нашлись: `DobSelect`, который **никогда** не мог
   собрать дату; ✕ диалога, перекрывавший «Готово» (подпись невозможно было сохранить);
   `defaultTheme="dark"`, из-за которого системная тема не работала **никогда**.
2. **Не верить своим же тестам.** Дважды «баг приложения» оказывался багом стаба: `**/…/users**`
   ловил и `/users/suggestions`, куда я слал `UserBriefDto` без `followedBy` — падал `RightSidebar`
   и уносил страницу. Я успел записать в роадмап неверное объяснение («гонка с dev-сервером»).
   Проверять на **production-сборке** (`next start`), dev отдаёт старый бандл.
3. **Готовый хук ≠ проведённый endpoint.** Карта врала именно на этом; теперь считает только то,
   до чего доберётся человек.
4. **Если бэкенд чего-то не умеет — не подделывать.** Так и написано в роадмапе: звонок не сделан,
   соавторов нет, миниатюры в уведомлении нет. Кнопка, которая называется одним, а делает другое,
   уже была («Показать архив» вела в «Сохранённое») — это хуже, чем её отсутствие.
5. **Скриншот — источник правды по вёрстке.** Метрики auth сняты canvas-сканом границ (DPR 1.25):
   инпут 60px, кнопка 44px, колонка 560px. На глаз было 56/48/700.
